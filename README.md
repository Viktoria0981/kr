# Инструкция: автоматическое подключение библиотеки

## Как сделать, чтобы библиотека "зарегистрировала" себя в системе вашего пользователя

### Шаг 1: Подготовка библиотеки (добавление «паспорта»)

Откройте `CMakeLists.txt` проекта библиотеки. Добавьте инструкции по установке в самый конец файла.

**Важно:** Посмотрите, как называется цель библиотеки в команде `add_library(...)`. Допустим, она называется `my_math_lib`.

Добавьте этот код (замените `my_math_lib` на реальное название):

```cmake
# 1. Говорим, куда копировать файлы при установке
install(TARGETS my_math_lib 
        EXPORT MyLibTargets            # Метка для экспорта
        LIBRARY DESTINATION lib        # Библиотеки (.so) -> в папку lib
        ARCHIVE DESTINATION lib        # Статические библиотеки -> в папку lib
        INCLUDES DESTINATION include)  # Где искать заголовки

# 2. Копируем заголовочные файлы (замените на свои .h файлы)
install(FILES my_math_lib.h DESTINATION include)

# 3. Генерируем конфиг-файл (тот самый "паспорт")
# Чтобы find_package(MyLib) в приложении сработал
install(EXPORT MyLibTargets
        FILE MyLibConfig.cmake
        NAMESPACE MyLib::
        DESTINATION lib/cmake/MyLib)

Шаг 2: Установка библиотеки в локальную директорию
Теперь нужно «установить» библиотеку в вашу домашнюю папку (~/.local), чтобы не использовать sudo. Выполните в терминале, находясь в папке библиотеки:

bash
# 1. Конфигурируем сборку и указываем путь установки в ~/.local
cmake -B build -DCMAKE_INSTALL_PREFIX=$HOME/.local

# 2. Компилируем
cmake --build build

# 3. Устанавливаем (файлы разлетятся по папкам в ~/.local)
cmake --install build
Шаг 3: Настройка "автозапуска" для системы (через .bashrc)
Чтобы приложение видело библиотеку при сборке и находило её при запуске без лишних команд, настройте ваш профиль пользователя.

Откройте файл конфигурации:

bash
nano ~/.bashrc
Пролистайте в самый низ и вставьте эти две строки:

bash
# Чтобы CMake находил установленные конфиги библиотек
export CMAKE_PREFIX_PATH="$HOME/.local:$CMAKE_PREFIX_PATH"

# Чтобы приложение находило динамическую библиотеку (.so) при запуске
export LD_LIBRARY_PATH="$HOME/.local/lib:$LD_LIBRARY_PATH"
Сохраните и выйдите: Ctrl+O → Enter → Ctrl+X

Примените настройки прямо сейчас:

bash
source ~/.bashrc
Шаг 4: Сборка и запуск приложения
Теперь вы можете идти в папку приложения. Вам не нужно ничего менять в его файлах. Просто запускайте стандартные команды:

bash
# Переходим в папку приложения
cd /path/to/app_source

# 1. Конфигурируем (теперь find_package сработает автоматически!)
cmake .

# 2. Собираем
cmake --build .

# 3. Запускаем
./имя_твоего_приложения
Почему это сработает после релогина?
Файлы библиотеки физически лежат в вашем $HOME/.local

Переменные CMAKE_PREFIX_PATH и LD_LIBRARY_PATH прописаны в .bashrc, а этот файл выполняется автоматически каждый раз, когда вы открываете терминал или входите в систему

Приложение при сборке через find_package() заглянет в CMAKE_PREFIX_PATH, увидит там ваш MyLibConfig.cmake и узнает, где лежат заголовки и сама библиотека
